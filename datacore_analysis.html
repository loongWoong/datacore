数据中台项目架构分析报告

项目概述

本项目是一个完整的数据中台解决方案，包含了数据资产管理、元数据管理、数据血缘追踪、数据质量监控、报表生成和任务调度等多个核心功能模块。该项目采用微服务架构，各服务之间通过网关统一接入，前端使用React构建。

1. 网关服务 (Gateway Service)

功能特性 技术栈 路由配置

• 请求/响应过滤
• 跨域处理

• /api/lineage/** -> 端口8081

2. 元数据服务 (Metadata Service)

负责管理所有数据资产的元数据信息，包括表结构信息、字段信息、业务属性等。

功能模块 技术栈 主要接口
• 表管理：表元数据的增删改查
• 字段管理：字段元数据的增删改查
• 版本管理：元数据版本控制 		• Spring Boot 3.x + MyBatis Plus
• MySQL 8.x
• Redis 7.x

• POST /api/metadata/tables - 创建表元数据
• GET /api/metadata/columns/table/{tableId} - 获取表的字段列表
• GET /api/metadata/versions/table/{tableId} - 获取表的版本列表

3. 数据血缘服务 (Lineage Service)

负责管理和展示数据在各个处理环节之间的血缘关系，通过解析dbt模型和SQL语句自动构建数据血缘图谱。

功能模块 技术栈 主要接口
• 血缘关系构建：支持dbt模型和SQL语句的血缘关系自动构建
• 血缘关系查询：支持上下游血缘关系查询和可视化展示
• 图谱展示：提供血缘图谱的可视化展示功能 		• Spring Boot 3.x
• Neo4j 5.x (图数据库)
• MySQL 8.x
• Redis 7.x
• JSqlParser (SQL解析)
• SpringDoc Swagger UI 		• GET /api/lineage/graph - 获取血缘关系图
• GET /api/lineage/upstream/{tableName} - 获取上游血缘关系
• POST /api/lineage/build/dbt - 触发dbt血缘构建

总结

本数据中台项目采用了现代化的微服务架构，具有以下特点：

  1. 模块化设计：各服务职责清晰，独立部署，便于维护和扩展。
  2. 多样化技术栈：根据不同业务需求选择了合适的技术组件，如Neo4j用于血缘关系存储，Elasticsearch用于资产搜索等。
  3. 统一网关入口：通过网关服务统一管理API访问，提供了良好的安全性和可管控性。
  4. 完善的存储体系：结合关系型数据库、缓存、搜索引擎、图数据库等多种存储方案，满足不同场景需求。
  5. 前端分离架构：采用前后端分离设计，前端使用现代化的React框架，用户体验良好。

整体而言，该项目具备了企业级数据中台应有的核心功能，能够有效支撑企业的数据资产管理、数据质量保障、数据分析应用等业务需求。
